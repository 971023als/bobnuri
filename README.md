# 🛡️ BoB누리 취약점 실습 프로젝트 (Bobnuri) - 모의 해킹 및 방어
BoB누리 실습 프로젝트에 오신 것을 환영합니다! 이 프로젝트는 **모의 해킹 실습 및 방어**를 위한 학습용 웹 애플리케이션으로, 단계별 보안 수준 설정을 통해 다양한 취약점과 대응 방안을 실습할 수 있도록 설계되었습니다.

## 📊 보안 단계별 설정 (Difficulty Levels)

학습자는 상단 메뉴의 **보안 설정 메뉴**를 통해 실시간으로 보안 단계를 변경하며 실습할 수 있습니다. 각 단계는 실제 환경(Docker 또는 XAMPP)의 환경변수나 세션을 통해 관리됩니다.

- **`Level 1 (Beginner)`**: **완전 취약 상태**
  - **보안 기능**: 없음, 평문 비밀번호(Plaintext) 저장
  - **학습 포인트**: 기초적인 SQL Injection (`' OR '1'='1'`) 등을 통한 인증 우회 실습.
- **`Level 2 (Easy)`**: **기초 방어 적용**
  - **보안 기능**: `mysqli_real_escape_string()`을 통한 특수문자 필터링
  - **학습 포인트**: 기본적인 인젝션 차단 확인 및 필터링을 우회할 수 있는 다른 취약점 탐색.
- **`Level 3 (Medium)`**: **취약한 암호화 적용**
  - **보안 기능**: SQL 필터링 + **MD5 해시**를 이용한 비밀번호 암호화
  - **학습 포인트**: 노출된 해시값을 Rainbow Table이나 사전 공격(Dictionary Attack)으로 복호화하는 사례 학습.
- **`Level 4 (Hard)`**: **고급 보안 적용**
  - **보안 기능**: **Prepared Statements** (매개변수화 쿼리) + **Bcrypt** 강력한 해싱.
  - **학습 포인트**: 전문가 수준의 보안 기술이 어떻게 공격을 완벽하게 차단하는지 확인 (SQLi 불가능).
- **`Level 5 (Secure)`**: **최종 방어 단계**
  - **보안 기능**: Level 4의 모든 보안 + **XSS 방어(`htmlspecialchars`)** 및 입출력 검증 강화.
  - **학습 포인트**: 스크립트 삽입(XSS) 공격 시도가 어떻게 무력화되는지 확인하며 완벽한 보안 설계 학습.

## 🚀 설치 및 환경 구축 가이드

### 방법 1: Docker 사용 (권장 및 가장 빠름)
가장 간편하게 전체 환경을 구축하고 다른 기기에서도 접속할 수 있는 방법입니다.

1. **저장소 복제**:
   ```bash
   git clone https://github.com/your-username/bobnuri.git
   cd bobnuri
   ```
2. **컨테이너 실행**:
   ```bash
   docker-compose up -d --build
   ```
3. **로컬 접속**: [http://localhost:8080](http://localhost:8080)
4. **외부 기기(모바일 등) 접속**:
   - **로컬 네트워크**: 같은 Wi-Fi에 연결된 기기에서 `http://[PC_IP]:8080`으로 접속합니다.
   - **외부 인터넷 접속 (포트 포워딩) 가이드 (Docker 사용)**:
     1. **내부 IP 및 MAC 주소 고정**: `ipconfig /all` 명령으로 확인한 물리적 주소(MAC)를 공유기 설정의 [DHCP 고정 할당]에 등록하여 내부 IP(예: `192.168.0.100`)가 바뀌지 않게 고정합니다.
     2. **포트 포워딩 설정**: 
        - 외부 포트: `8080` (또는 보안을 위해 `18080`, `28080` 등 높은 번호 추천)
        - 내부 포트: `8080` (도커 컨테이너 서비스 포트)
        - 내부 IP: 고정한 PC의 IPv4 주소
     3. **Windows 방화벽 사전 개방**: `wf.msc` 실행 > `인바운드 규칙` > `새 규칙` > `포트` > `TCP` > `8080` 허용 > 모든 프로필 체크 > 이름 `BoBnuri-Docker`로 저장.
     4. **NAT Loopback 미지원 시 테스트**: 같은 Wi-Fi 망에서는 외부 IP 접속이 안 될 수 있으므로 반드시 **스마트폰 LTE/5G 환경**에서 `http://[공인IP]:8080`으로 접속 테스트를 진행합니다.
     5. **포트 개방 확인 및 DDNS**: `yougetsignal.com`에서 포트가 `Open`인지 확인하고, 유동 IP 환경이라면 공유기의 DDNS 기능을 이용하여 `아이디.iptime.org:8080` 주소를 생성합니다.
     6. **Troubleshooting**: 만약 접속이 안 된다면 통신사에서 포트를 차단했을 수 있으므로 외부 포트를 다른 번호로 바꿔보거나 임시적으로 **DMZ 설정**을 사용할 수 있으나 보안상 권장하지 않습니다.
     - **⚠️ 보안 필독**: 이 테스트는 모의 해킹용으로 매우 취약합니다. **실습 직후 반드시 공유기 규칙을 삭제하고 도커 컨테이너를 종료**하십시오.

### 방법 2: Windows XAMPP 환경 (로컬 도메인 연동)
실제 도메인(`mock-bobnuri.com`)을 사용하여 실제 운영 환경처럼 실습하고 싶을 때 유용합니다.

1. **사전 준비**: XAMPP(Apache, MySQL)가 설치되어 있어야 합니다.
2. **자동화 스크립트 실행**:
   - 프로젝트 루트 폴더에서 **`configure_apache.ps1`** 파일을 마우스 오른쪽 버튼으로 클릭 후 **[PowerShell에서 실행]**을 선택합니다.
   - **관리자 권한(UAC)** 창이 뜨면 **[예]**를 누릅니다.
   - **스크립트가 자동으로 수행하는 작업**:
     - Windows `hosts` 파일에 `127.0.0.1 mock-bobnuri.com` 등록
     - 아파치 `httpd-vhosts.conf`에 가상 호스트 설정 추가
     - 아파치 메인 설정(`httpd.conf`)에서 가상 호스트 기능 활성화
     - DNS 캐시 초기화 및 아파치 서비스 재시작
3. **접속 주소**: [http://mock-bobnuri.com](http://mock-bobnuri.com)
4. **외부 기기 접속 (로컬/외부)**:
   - **로컬 네트워크**: 같은 Wi-Fi에 연결된 기기에서 `http://[PC_IP]`로 접속합니다.
   - **외부 인터넷 접속 (포트 포워딩) 가이드 (XAMPP 사용)**:
     1. **아파치 포트 충돌 확인**: XAMPP `Start` 실패 시 `Netstat` 버튼을 눌러 `80` 포트 점유 프로그램(예: IIS, Skype)을 종료하거나 `httpd.conf`에서 `Listen 80` -> `8081`로, `httpd-ssl.conf`에서 `443` -> `4433`으로 변경합니다.
     2. **내부 IP 및 MAC 주소 고정**: `ipconfig /all`로 확인한 MAC 주소를 공유기 설정의 [DHCP 고정 할당]에 등록하여 내부 IP(예: `192.168.0.50`)가 바뀌지 않게 고정합니다.
     3. **공유기 포트 포워딩 설정**: 
        - **외부/내부 포트**: `80` (또는 `8081`) / **내부 IP**: 고정한 PC IP
        - 포트 포워딩이 복잡하다면 **DMZ 설정**을 통해 PC의 모든 포트를 일시 개방할 수 있으나 보안상 **절대 비권장**합니다.
     4. **Windows 방화벽 인바운드 허용**: `wf.msc` 실행 > `인바운드 규칙` > `새 규칙` > `포트` > `80, 8081` 허용 규칙을 추가하여 인입을 차단 해제합니다.
     5. **공인 IP 및 DDNS 주소**: 네이버에서 '내 아이피'를 검색해 확인하거나 iptime의 DDNS 기능을 사용하여 `아이디.iptime.org` 주소를 생성합니다.
     6. **외부 접속 검증**: 스마트폰 LTE 환경에서 `http://[DDNS주소]:[포트]`를 입력해 접속합니다.
     - **⚠️ 보안 초특급 주의**: 이 사이트는 공격 실습용으로 매우 취약합니다. **실습이 끝나면 공유기 규칙을 즉시 삭제하고 아파치를 종료**하십시오.
5. **참고**: XAMPP 환경에서 데이터베이스 기본값은 프로젝트 루트의 `.env` 파일에서 설정 가능하며, 웹 페이지 상단 메뉴에서도 실시간 변경이 가능합니다.

## 📂 프로젝트 구조
- `configure_apache.ps1`: XAMPP/Apache 환경을 위한 원클릭 자동 설정 스크립트.
- `security_config.php`: 보안 레벨 제어 및 `.env` 로드 핵심 로직.
- `change_security.php`: 웹 UI에서 보안 레벨을 즉시 변경하는 핸들러.
- `db/init.sql`: 도커 실행 시 자동 생성되는 DB 스키마 및 샘플 데이터.
